Промт: Реализация загрузки изображений (Fullstack Node.js + React)
Контекст системы:
У меня есть Fullstack приложение (Vite + React на фронтенде, Node.js + Express + PostgreSQL на бэкенде). Архитектура использует контроллеры, роуты и кастомные хуки для управления состоянием профиля пользователя.

Задача:
Реализовать функционал загрузки аватара пользователя с сохранением файла на сервере и записи пути в БД.

Технические требования:

Бэкенд (Node.js/Express):

Использовать библиотеку Multer для обработки multipart/form-data.

Настроить diskStorage: файлы должны сохраняться в src/server/uploads/ с уникальными именами (timestamp + random).

Контроллер обновления: Должен принимать req.file, формировать полный URL (например, http://localhost:5000/uploads/filename) и обновлять поле avatar_url в таблице users через SQL UPDATE ... RETURNING.

Статика: Настроить express.static в index.ts, чтобы папка uploads была доступна извне по пути /uploads.

Auth API: Роут получения данных текущего пользователя (GET /me) должен обязательно возвращать поле avatar_url из базы.

Фронтенд (React + TypeScript):

Интерфейсы: Обновить interface UserData, добавив необязательное поле avatar_url?: string.

Хук профиля: В useProfile обеспечить получение данных из API и синхронизацию с localStorage.

Компоненты (Header/Profile): Реализовать логику отображения:

Если user.avatar_url существует — рендерить <img> с object-fit: cover.

Если нет — рендерить "Fallback" (первую букву имени).

Добавить обработку onError для тега img, чтобы скрыть битую картинку и показать букву.
